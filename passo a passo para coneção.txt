Resposta:
Para integrar o seu formulário HTML com o banco de dados MySQL usando Python, a melhor abordagem é utilizar o **Flask**, um micro-framework web em Python que é muito simples de configurar no VS Code, junto com o conector `mysql-connector-python`.

Como você mencionou que está desenvolvendo um material de estudo/apostila, organizei a estrutura do projeto de forma limpa e profissional.

## 3. O Código Python (`app.py`)

Este script cria o servidor web local, recebe os dados do HTML e faz a query `INSERT` no MySQL.

```python
from flask import Flask, render_template, request, redirect
import mysql.connector

app = Flask(__name__)

# Configuração da conexão com o banco de dados
def conectar_banco():
    return mysql.connector.connect(
        host="localhost",
        port=3306,
        user="root",
        password="root",
        database="apostas"
    )

# Rota principal: renderiza o formulário HTML
@app.route('/')
def index():
    return render_template('formulario.html')

# Rota que processa o envio dos dados do formulário
@app.route('/inserir', methods=['POST'])
def inserir_cliente():
    if request.method == 'POST':
        # Captura os dados vindos do formulário HTML usando o atributo 'name'
        nome = request.form['nome']
        email = request.form['email']
        nasc = request.form['nasc']
        celular = request.form['celular']
        cpf = request.form['cpf']
        senha = request.form['senha']
        
        try:
            # Conecta ao banco e prepara o cursor
            conexao = conectar_banco()
            cursor = conexao.cursor()
            
            # Comando SQL adaptado exatamente para a sua tabela
            sql = """INSERT INTO cliente (Nome, Email, Nasc, Celular, CPF, Senha) 
                     VALUES (%s, %s, %s, %s, %s, %s)"""
            valores = (nome, email, nasc, celular, cpf, senha)
            
            cursor.execute(sql, valores)
            conexao.commit() # Confirma a inserção no banco
            
            cursor.close()
            conexao.close()
            
            return "<h3>Cliente cadastrado com sucesso!</h3> <a href='/'>Voltar</a>"
            
        except mysql.connector.Error as erro:
            return f"<h3>Erro ao inserir no banco de dados: {erro}</h3> <a href='/'>Voltar</a>"

if __name__ == '__main__':
    # Roda o servidor no modo debug (reinicia automaticamente ao salvar alterações)
    app.run(debug=True)

```

## 4. Passo a Passo para Executar no VS Code

Siga esta sequência para garantir que as dependências e o ambiente funcionem perfeitamente:

1. **Abrir a pasta no VS Code:** Pré-requisito.
Abra o VS Code, vá em **File > Open Folder** (Arquivo > Abrir Pasta) e selecione a pasta onde você salvou os arquivos estruturados acima.


2. **Abrir o Terminal Integrado:** No VS Code.
No menu superior do VS Code, clique em **Terminal > New Terminal**. Ele abrirá um terminal na parte inferior da tela, já apontando para a pasta do seu projeto.


3. **Instalar as bibliotecas necessárias:** Terminal.
Digite o seguinte comando no terminal para instalar o Flask e o conector do MySQL e pressione Enter:

```bash
pip install Flask mysql-connector-python

```

*Aguarde a conclusão da instalação das duas ferramentas.*


4. **Certificar-se de que o Banco está ativo:** MySQL Workbench.
Abra o MySQL Workbench, certifique-se de que a database `apostas` existe e que a tabela `cliente` foi criada exatamente com o comando que você enviou.


5. **Executar o script Python:** Terminal.
De volta ao terminal do VS Code, execute o arquivo digitando:

```bash
python app.py

```

Você verá uma mensagem no terminal informando que o servidor está rodando, parecida com:
`* Running on [http://127.0.0.1:5000](http://127.0.0.1:5000)`


6. **Testar no Navegador:** Navegador.
Abra o seu navegador web (Chrome, Edge, Firefox, etc.) e acesse o endereço:
**[http://127.0.0.1:5000](http://127.0.0.1:5000)**
O formulário vai aparecer. Preencha os dados e clique em **Cadastrar** para testar a gravação no banco de dados.


---

> **Aviso de segurança importante:** Como este código é para fins de estudo/local, a senha está sendo gravada em texto puro (`varchar(11)`). Em sistemas de produção reais, as senhas nunca devem ser salvas assim; usa-se bibliotecas de criptografia (como `Werkzeug` ou `bcrypt`) para gerar um *hash* seguro e o campo no banco geralmente precisa de mais espaço (ex: `varchar(255)`).
